# 架装品実績レポート（500）のWEB化計画

## 概要
現在ローカルのElectron（デスクトップ）アプリとして稼働している「架装品実績レポート」について、**指定のスプレッドシートからデータを取得するWEBアプリケーション（React + Vite）**に移行します。完成後はGitHub（`kasouhin_500_Web`）にプッシュし、Cloudflareにデプロイできる状態に構成します。

## 実装ステップ

### 1. プロジェクトの初期化
- `E:\Dropbox\...500\web` フォルダ内に `Vite (React + JavaScript)` プロジェクトを作成します。
- `tailwindcss`, `lucide-react`, `recharts` など元アプリで使用していたパッケージをインストールし、同じデザイン・仕様が動く環境を再構築します。

### 2. コンポーネントおよびUIの移植
- 元の `App.jsx`, `utils/aggregator.js`, `index.css` などをWEB版にコピーします。
- Electron特有の処理（`window.electronAPI.loadExcelData`, `ipcRenderer` など）を削除し、WEBブラウザ内で動作するように書き換えます。
- PDF出力等の機能はブラウザの標準印刷機能（`window.print()`）にフォールバックされるよう調整します（元アプリに実装済みのため活用します）。

### 3. Google Apps Script (GAS) APIの構築
- データ連携用として、**Google Apps Script側でスプレッドシートのデータをJSONとして返すAPI** を構築します。
- データの形式はローカルアプリで読み込んでいた `Rows`, `Years`, `LeaseCompanies` 等と同形式にし、フロントエンド側でエラーが出ないよう処理します。
- Viteプロジェクト内に `Code.gs`（GAS用のスクリプト）を作成しご提示します。
- **※スプレッドシートの「拡張機能」>「Apps Script」にそのコードを貼り付け、「ウェブアプリとしてデプロイ」していただく必要があります。**

### 4. アプリケーションとAPIの接続
- Viteアプリ内から、手順3で発行したGASのWebアプリURL（エンドポイント）へFetchリクエストを送信し、データを取得する処理を実装します。

### 5. GitHub へのプッシュ設定
- 初期化完了後、指定されたリポジトリ `https://github.com/dezigozi/kasouhin_500_Web.git` へ自動的にコードを `git push` できるようにセットアップしコミット＆プッシュを実行します。

> GASを設定後、発行されたURLをReactアプリ側に埋め込む必要があります。環境変数（`.env`）から読み込むように準備しておきますので、私がコード作成後にご自身でGASをデプロイし、発番されたURLを共有いただく段取りでよろしいでしょうか？
> 
> 2. **GitHubへのプッシュ**
> こちらの環境（コンソール）から直接GitHubにプッシュするためには、認証情報（SSHキーやToken等）が正しくローカルにセットアップされている必要があります（通常は問題ありません）。もしエラーが出る場合は、ローカル環境へのソースコード作成まで行い、ご自身でPushしていただく形になりますが問題ないでしょうか？
